# איפה סודות אמיתיים שמורים — לא הערכים עצמם

הקובץ הזה מצביע לאן מפתחות/סיסמאות אמיתיים שמורים, בלי להכיל אותם. הסיבה: קובץ גלובלי עם ערכי סוד אמיתיים הופך לנקודת כשל יחידה — כל צ'ט קלוד קוד שיש לו גישה לריפו הזה מקבל גישה בפועל לכל האפליקציות בבת אחת, וגם git history שומר ערכים ישנים לצמיתות גם אחרי מחיקה. **אף פעם אל תוסיפו כאן ערך סוד אמיתי — רק שם/label ואיפה למצוא אותו.**

כלל אצבע לגבי מה פר-אפליקציה לעומת מה שיכול להיות משותף:

| סוג מפתח | פר-אפליקציה או ניתן לשיתוף | למה |
|---|---|---|
| Google Play upload keystore | **תמיד פר-אפליקציה, לעולם לא לשתף** | דליפה של keystore אחד לא צריכה לסכן את היכולת לחתום גרסאות לכל שאר האפליקציות |
| Firebase service account JSON | פר-פרויקט Firebase | כל אפליקציה = פרויקט Firebase נפרד עם הרשאות נפרדות |
| Apple Store Connect API Key (`.p8`) | לרוב פר-אפליקציה (כל אפליקציה = integration נפרד ב-Codemagic), אבל מפתח עם הרשאת App Manager ברמת ה-Team **יכול טכנית** לשמש כמה אפליקציות של אותו Team — לא הונח בפועל, לבדוק לפני הסתמכות | תלוי בהיקף ההרשאה שנבחר ביצירת המפתח ב-App Store Connect |
| APNs key (push notifications) | לרוב פר-אפליקציה | כל אפליקציה = Firebase Cloud Messaging נפרד |
